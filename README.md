# ระบบคลังงานวิจัย · Research Repository

เว็บแอปพลิเคชันคลังงานวิจัยและโครงงานของ **วิทยาลัยอาชีวศึกษาร้อยเอ็ด** สำหรับรวบรวม เผยแพร่ และสืบค้นงานวิจัยของครู โครงงานนักเรียนนักศึกษา สิ่งประดิษฐ์ และโครงงานวิทยาศาสตร์

พัฒนาด้วย **PHP 8** และ **MariaDB 10** (ไม่ต้องใช้ Composer หรือ build tool ใด ๆ) ออกแบบให้รันบน XAMPP ได้ทันที

> โค้ดต้นแบบการออกแบบ (จาก Claude Design) เก็บไว้ที่ [`project/`](project/) เพื่อใช้อ้างอิงหน้าตา UI

## ความสามารถ

- **เว็บสาธารณะ** — หน้าแรกพร้อมสถิติและงานวิจัยล่าสุด, สืบค้นขั้นสูง (คำค้น + กรองตามประเภท/สาขา/ปี + เรียงลำดับ), หน้ารายละเอียดพร้อมบทคัดย่อ คำสำคัญ ผู้จัดทำ และดาวน์โหลดเอกสารแยกบท
- **ส่วนผู้ดูแล** — แดชบอร์ดสถิติ, จัดการงานวิจัย (เพิ่ม/แก้ไข/เปลี่ยนสถานะ/ลบ/อนุมัติเผยแพร่), นำเข้างานวิจัยพร้อมอัปโหลด PDF แยกบท, จัดการประเภทงานวิจัย และจัดการผู้ใช้งาน
- แสดงเฉพาะงานวิจัยสถานะ **เผยแพร่** บนเว็บสาธารณะ; เอกสารแต่ละบทกำหนดได้ว่าเปิดเผยหรือปิด
- โหมดสว่าง/มืด/อัตโนมัติ, รองรับภาษาไทยเต็มรูปแบบ (utf8mb4)

## การติดตั้ง

ต้องมี **PHP ≥ 8.0** และ **MariaDB ≥ 10** (มากับ XAMPP อยู่แล้ว)

1. วางโปรเจกต์ไว้ใต้ `htdocs` ของ XAMPP (เช่น `C:\xampp\htdocs\rvc.arch`)
2. เปิดบริการ **Apache** และ **MySQL** จาก XAMPP Control Panel
3. ตั้งค่าฐานข้อมูล — ค่าเริ่มต้นใช้ MariaDB บน `127.0.0.1:3306` user `root` ไม่มีรหัสผ่าน หากต่างจากนี้ ให้แก้ [`config/config.php`](config/config.php)
4. ติดตั้งผ่านฟอร์ม — เปิด **`http://your-host/rvc.arch/install.php`** ในเบราว์เซอร์ แล้วกรอก:
   - การเชื่อมต่อฐานข้อมูล (host / port / ชื่อ DB / user / password) — ระบบจะทดสอบการเชื่อมต่อและสร้างฐานข้อมูลให้อัตโนมัติ
   - บัญชีผู้ดูแล (ชื่อ / อีเมล / รหัสผ่าน ≥ 8 ตัว)
   - ชื่อระบบและหน่วยงาน
   - เลือกว่าจะ “เพิ่มข้อมูลตัวอย่าง” หรือ “ติดตั้งแบบว่าง” สำหรับใช้งานจริง

   ฟอร์มจะเขียนไฟล์ [`config/config.php`](config/config.php) ให้อัตโนมัติตามค่าที่กรอก จึงใช้ติดตั้งบนเซิร์ฟเวอร์จริงได้เลย และ**ตรวจสอบ `mod_rewrite` / `.htaccess` ให้อัตโนมัติ** — ถ้ายังไม่ทำงานจะขึ้นคำเตือนพร้อมวิธีแก้ (เปิด `mod_rewrite`, ตั้ง `AllowOverride All`)
   > **หมายเหตุ:** การติดตั้งจะลบตารางเดิมทั้งหมดแล้วสร้างใหม่ (destructive) — ต้องติ๊กยืนยันก่อนดำเนินการ
   >
   > สำหรับนักพัฒนา: `php install.php` (CLI) จะใช้ค่าจาก `config/config.php` ที่มีอยู่ พร้อมข้อมูลตัวอย่างและผู้ดูแล `somchai@rvc.ac.th` / `admin1234`
5. เปิดใช้งานที่ **http://your-host/rvc.arch/** และ **ลบไฟล์ `install.php`** ออกหลังติดตั้งเสร็จ

## โครงสร้างโปรเจกต์

```
index.php            Front controller + ตารางเส้นทาง (routing)
install.php          ติดตั้ง/รีเซ็ตฐานข้อมูล + ข้อมูลตัวอย่าง
config/              การตั้งค่า (config.php เก็บ credential, ถูก gitignore)
app/                 App, Database, Auth, Repository (ชั้นเข้าถึงข้อมูล SQL), helpers
app/controllers/     Public / Auth / Admin controllers
views/               เทมเพลต PHP (layout, partials, public, admin, auth)
assets/              app.css (design tokens), theme.js, admin.js
sql/schema.sql       โครงสร้างตาราง (utf8mb4)
storage/uploads/     ไฟล์ PDF ที่อัปโหลด (เข้าถึงผ่าน /download เท่านั้น)
project/             ต้นแบบการออกแบบจาก Claude Design (อ้างอิง UI)
```

## ความปลอดภัย

- ป้องกัน CSRF ในทุกฟอร์ม POST, รหัสผ่านเก็บแบบ hash (`password_hash`)
- ไฟล์ที่อัปโหลดเสิร์ฟผ่านเส้นทาง `/download/{id}` เท่านั้น เพื่อบังคับสิทธิ์เปิดเผย/ปิด — เข้าถึงไดเรกทอรีโดยตรงถูกปฏิเสธ (403)
- เข้าถึง `config/`, `app/`, `sql/`, `views/` โดยตรงผ่านเว็บถูกปฏิเสธ

ก่อนนำขึ้นใช้งานจริง: ตั้งรหัสผ่าน MariaDB, ลบหรือจำกัดสิทธิ์ `install.php`, และเปลี่ยนรหัสผ่านผู้ดูแลเริ่มต้น
